# /note-3-check

完成した記事の品質チェックを実行するスキル。

---

## 実行手順

### Step 1: 記事ファイルの指定

ユーザーに確認する:
- `articles/` 配下の記事一覧を表示
- チェック対象の記事を選択してもらう

または、直前に `/note-2-write` で生成した記事を自動で対象にする。

### Step 2: 文字数基準の確認

ユーザーに確認する:
```
文字数の基準を選んでください:
1. 2,000〜3,000字（ベース）
2. 3,000〜5,000字（中〜長め）
3. 5,000字以上（長文ガイド）
4. カスタム（数値を直接指定）
```

### Step 3: 品質チェック実行

選択に応じてオプションを設定し、`tools/note-api/quality_check.py` を実行する:

```bash
python tools/note-api/quality_check.py articles/{slug}/article.md [--min-chars N --max-chars N]
```

- 1（ベース）: `--min-chars 2000 --max-chars 3000`
- 2（中〜長め）: `--min-chars 3000 --max-chars 5000`
- 3（長文）: `--min-chars 5000 --max-chars 8000`
- 4: `--min-chars {指定} --max-chars {指定}`

### Step 4: 結果確認

チェック結果を表示する。全項目合格であればそのまま完了。

不合格項目がある場合:
1. 不合格項目の一覧を表示
2. 各項目について修正方針を提示
3. ユーザーの承認を得て修正
4. 修正後に再チェック（全項目合格するまで繰り返す）

### Step 5: 修正・再チェック

修正は以下の優先度で行う:
1. **段落文数** — 4文以上の段落を分割（最重要）
2. **文字数** — 目標範囲に収める
3. **です・ます調** — 常体の文末を修正
4. **note非対応記法** — テーブル等を箇条書きに変換
5. **絵文字** — 削除
6. **その他** — 個別対応

修正後に `article.md` を上書き保存し、再度 quality_check.py を実行する。

---

## 注意事項
- 品質チェックはnote入稿前に必ず実行する
- `/note-start`（一括実行）でも自動的にこのステップが実行される
- 修正時に見出し構造は変更しない
