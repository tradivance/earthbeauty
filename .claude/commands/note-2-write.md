# /note-2-write

承認済みの構成に沿ってSEO記事の本文を執筆するスキル。

---

## 実行手順

### Step 1: 設定読み込み

以下のファイルを**すべて**読み込む:
- `config/target.md`（ターゲット像）
- `config/writing-rules.md`（ライティングルール）
- `config/tone.md`（トーン＆マナー）
- `config/article-goal.md`（記事のゴール）
- `knowledge-base/` 配下の全 `.md` ファイル

### Step 2: 構成読み込み

ユーザーに記事を指定してもらい、`articles/{slug}/structure.md` を読み込む。
また、`data/keyword-groups.csv` から対応するKW群を再読み込みする。

構成ファイルがない場合:
- 「先に `/note-1-plan` で構成を作成してください」と案内する

### Step 3: リサーチ

WebSearchで記事テーマについて追加調査する:
- 具体的な事例・データ・統計
- 最新情報・トレンド
- knowledge-base/ にない補足情報

### Step 4: 記事執筆

以下のルールを**厳守**して執筆する:

1. **構成は絶対に変更しない** — 承認済みのH2/H3をそのまま使う
2. **ライティングルール遵守** — `config/writing-rules.md` の全ルールに従う
3. **トーン再現** — `config/tone.md` と knowledge-base/ の口調を再現する
4. **文字数達成** — structure.md に記載された目標文字数を達成する
5. **Markdown形式** — note.com用のMarkdownで出力
6. **画像プレースホルダー** — `{画像: 説明}` 形式でプレースホルダーを配置

執筆ルール:
- です・ます調で統一
- 1段落は1〜3文まで（4文以上は段落分割）
- 各セクションに箇条書きを活用（全体が箇条書きだらけにならないよう注意）
- 太字（`**...**`）は1文単位で使う
- 絵文字は使わない
- テーブル記法は使わない（noteで表示崩れるため）

### Step 5: KW配置の確認

本文確認時に**必ずKW一覧を表示**する:
```
| キーワード | 月間Vol | トレンド | 配置場所 |
|-----------|---------|---------|---------|
```

### Step 6: 文字数計測（必須）

執筆完了後、**自己申告の文字数を信用せず**、必ずツールで実測する:
1. `articles/{slug}/article.md` に一旦保存
2. `python tools/note-api/quality_check.py` またはPythonの `len()` で純テキスト文字数を計測
3. 計測結果をユーザーに共有
4. 目標文字数の範囲外であれば調整

### Step 7: 出力・保存

文字数確認後、完成した記事を画面に表示し、ユーザーの確認を得る。
承認後、`articles/{slug}/article.md` を最終版として保存する。

---

## 注意事項
- knowledge-base/ の過去記事に具体例や数字があれば積極的に引用する
- 記事の最後にCTAが必要な場合は `config/article-goal.md` の内容に沿って設置
- 文字数が目標に届かない場合は具体例・補足で充実させる（水増しNG）
