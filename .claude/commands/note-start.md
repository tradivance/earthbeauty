# /note-start

構成作成→記事執筆→品質チェック→note入稿を一気通貫で実行するスキル。

---

## 実行手順

### Step 0: 記事番号の選択

ユーザーに確認する:
```
作成する記事を選んでください（番号で指定）:

data/keyword-groups.csv から記事一覧を表示
```

CSVが空の場合は、ユーザーにキーワードを直接入力してもらう。

### Step 1: 設定読み込み

以下のファイルを**すべて**読み込む:
- `data/keyword-groups.csv` から選択した記事#のKW群を抽出
- `config/target.md`（ターゲット像）
- `config/writing-rules.md`（ライティングルール）
- `config/tone.md`（トーン＆マナー）
- `config/article-goal.md`（記事のゴール）
- `knowledge-base/` 配下の全 `.md` ファイル（口調・表現の学習用）

### Step 2: キーワード確認・経験値確認

1. CSVから読み込んだKW群を表示する:
```
| キーワード | 月間Vol | トレンド | 記事内での役割 |
|-----------|---------|---------|--------------|
```

2. テーマの経験値・理解度をユーザーに確認する:
```
このテーマについて、あなたの経験値・理解度を教えてください:
1. 詳しい（実務経験あり・専門知識がある）
2. ある程度知っている（記事や書籍で学んだ・多少の経験あり）
3. あまり詳しくない（興味はあるが深い知識はない）
```

### Step 2.5: 目標文字数の確認

```
目標文字数はどのくらいにしますか？
1. 2,000〜3,000字（ベース）
2. 3,000〜5,000字（中〜長め）
3. 5,000字以上（長文ガイド）
```

### Step 3: 構成作成（= note-1-plan）

1. WebSearchで競合分析・関連キーワード調査
2. CSVの「記事内での役割」列を参考に、H2/H3の見出し構成を設計
3. 各H2に `（KW: ...）` メモで対応KWを明記
4. 構成提案時に**必ずKW一覧を表示する**:
```
| キーワード | 月間Vol | トレンド | 主な配置 |
|-----------|---------|---------|---------|
```
5. ユーザーの承認を得る
6. `articles/{slug}/structure.md` に保存

### Step 4: 記事執筆（= note-2-write）

1. WebSearchで追加リサーチ（事例・データ・統計）
2. 承認済み構成に沿って記事本文を執筆
   - 構成は絶対に変更しない
   - `config/writing-rules.md` と `config/tone.md` を厳守
   - 目標文字数を達成
3. 本文確認時に**必ずKW一覧を再表示**する
4. `articles/{slug}/article.md` に保存

### Step 4.5: 文字数計測（必須・自動実行）

執筆完了後、**ユーザー確認なしに自動実行**する:
1. `python tools/note-api/quality_check.py articles/{slug}/article.md` またはPythonの `len()` で純テキスト文字数を計測
2. **計測結果をユーザーに共有する**
3. 目標文字数の範囲外であれば自動調整し、再計測

### Step 5: 品質チェック（= note-3-check）

1. `python tools/note-api/quality_check.py articles/{slug}/article.md [--min-chars N --max-chars N]` を実行
2. 不合格項目があれば自動修正
3. 再チェックして全項目合格を確認

### Step 5.9: 入稿前の最終確認（必須・ユーザーOK必要）

品質チェック合格後、**必ずユーザーに以下を提示してOKをもらう**:

1. **記事ファイルのフルパス**を明示
2. **狙いKW一覧**を表示
3. ユーザーから「OK」をもらった後に Step 6 へ進む

### Step 6: ハッシュタグ提案・入稿確認

1. タイトルを提示し、変更があるか確認
2. 記事の内容を分析し、**ハッシュタグ候補を20個** `article.md` の末尾にコードブロックで追記:
   - 記事のメインテーマに直結するタグ（5〜7個）
   - 関連する周辺テーマのタグ（5〜7個）
   - note.comで検索されやすい一般的なタグ（3〜5個）
   - ニッチだが刺さる層がいるタグ（2〜3個）

### Step 7: note.com入稿（= note-4-publish）

1. `python tools/note-api/save_draft.py articles/{slug}/article.md --title "タイトル" --hashtags "tag1,tag2"` を実行
2. 成功したら編集URLを表示
3. 失敗したらエラー内容を報告

---

## 注意事項
- 一括実行でも、**構成の承認**と**入稿前の確認**では必ずユーザーに確認する
- 途中でエラーが発生した場合は、そのステップで止めてユーザーに報告する
- 品質チェックの自動修正は3回まで。3回で合格しない場合はユーザーに相談する
