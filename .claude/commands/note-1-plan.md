# /note-1-plan

SEO記事の見出し構成を作成するスキル。CSVからKW群を読み込み、構成を策定する。

---

## 実行手順

### Step 1: 設定読み込み

以下のファイルを**すべて**読み込む:
- `data/keyword-groups.csv`（KWグルーピング）
- `config/target.md`（ターゲット像）
- `config/writing-rules.md`（ライティングルール）
- `config/tone.md`（トーン＆マナー）
- `config/article-goal.md`（記事のゴール）
- `knowledge-base/` 配下の全 `.md` ファイル

### Step 2: 記事番号・キーワード受け取り

記事番号を受け取り、CSVから対応するKW群を抽出して表示:
```
| キーワード | 月間Vol | トレンド | 記事内での役割 |
|-----------|---------|---------|--------------|
```

または、CSV外のキーワードを直接入力してもOK。

### Step 3: テーマの経験値・理解度の確認

```
このテーマについて、あなたの経験値・理解度を教えてください:
1. 詳しい（実務経験あり・専門知識がある）
2. ある程度知っている（記事や書籍で学んだ・多少の経験あり）
3. あまり詳しくない（興味はあるが深い知識はない）
```

### Step 3.5: 目標文字数の確認

```
目標文字数はどのくらいにしますか？
1. 2,000〜3,000字（ベース）
2. 3,000〜5,000字（中〜長め）
3. 5,000字以上（長文ガイド）
```

### Step 4: リサーチ

WebSearchで以下を調査する:
1. キーワードの検索上位記事（競合分析）
2. 関連キーワード・サジェスト
3. 読者が知りたいこと（Q&Aサイト等）

### Step 5: 構成提案

CSVの「記事内での役割」列を参考に、H2/H3構成を設計する:

```markdown
# {記事タイトル案}

## {H2見出し1}（KW: シェアサロン 東京）
### {H3見出し1-1}

## {H2見出し2}（KW: レンタルサロン 渋谷）
...

---
想定文字数: X,XXX字
ターゲット: {config/target.mdの要約}
記事のゴール: {config/article-goal.mdの要約}
```

構成提案時に**必ずKW一覧を表示する**:
```
| キーワード | 月間Vol | トレンド | 主な配置 |
|-----------|---------|---------|---------|
```

### Step 6: 承認・保存

ユーザーの承認を得たら、`articles/{slug}/structure.md` に保存する。

---

## 注意事項
- 構成が承認されたら、以降のステップで見出し構造を変更しない
- CSVの「記事内での役割」はガイドライン。実際の構成は検索意図と競合分析に基づいて柔軟に調整してよい
- knowledge-base/ の素材から使えそうなネタがあればメモしておく
